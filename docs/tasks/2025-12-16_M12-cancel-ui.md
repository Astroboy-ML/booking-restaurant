# Ticket: M12 — Espace restaurateur : annuler/mettre à jour les réservations

## But
Permettre au restaurateur de consulter et annuler les réservations depuis une vue dédiée (gestion du flux MVP côté restaurant).

## Scope
- apps/web/ (composants/pages restaurateur)
- docs/DEV.md (notes d’usage)

## Contraintes
- Réutiliser l’API existante `DELETE /reservations/{id}` (pas de nouvelle route backend).
- Afficher les états (par ex. confirmée/annulée) si disponibles ; gérer erreurs 404/500.
- Confirmation d’action (dialogue ou double-clic minimal) ; pas d’auth encore.

## Deliverables
- Page “Restaurateur” listant les réservations avec action d’annulation par ligne.
- Rafraîchissement de la liste après suppression (sans recharger la page) et visibilité de l’état annulé.
- Message clair en cas d’erreur (ex: réservation déjà supprimée ou inexistante).
- Documentation courte dans la section front décrivant l’accès à l’espace restaurateur.

## Critères d’acceptation
- [ ] Cliquer “Annuler” côté restaurateur supprime la réservation via l’API et met à jour l’affichage.
- [ ] Les erreurs API sont affichées à l’utilisateur (404/500/validation).
- [ ] Aucun secret ou URL hardcodée dans le code.
- [ ] Documentation front mise à jour.

## Plan proposé
1) Ajouter un handler de suppression dans le client API et l’utiliser dans la page restaurateur.
2) Exposer la liste avec actions d’annulation et un retour visuel d’état (ex: tag annulé).
3) Tester manuellement et documenter les comportements d’erreur.
