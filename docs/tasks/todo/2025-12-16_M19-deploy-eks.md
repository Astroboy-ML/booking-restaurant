# Ticket: M19 Déploiement application sur EKS

## But
Déployer l’API (et si possible le web) sur EKS en s’appuyant sur les images ECR.

## Scope
- Manifests Kubernetes (ou Helm) adaptés à EKS
- Ingress/Service/ConfigMap/Secret
- Documentation de déploiement

## Contraintes
- Pas de secrets en clair ; utiliser Secrets/SSM/SealedSecrets si applicable
- Configurable pour différents environnements (dev/preprod)
- Healthchecks (readiness/liveness) configurés

## Deliverables
- Manifests/Helm chart pour l’API (et web si inclus)
- Documentation de déploiement (kubectl/Helm)
- Références aux images ECR

## Critères d’acceptation
- [ ] Un manifest/Helm chart permet de déployer l’API sur EKS avec readiness/liveness
- [ ] Les images ECR sont référencées via tag/sha
- [ ] Les secrets/config sont injectés sans être commités en clair

## Plan proposé
1) Préparer les manifests/Helm (Deployment, Service, Ingress, config)
2) Référencer les images ECR et variables nécessaires
3) Documenter les commandes de déploiement
