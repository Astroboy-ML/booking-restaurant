Organisation des tickets
- todo : ticket cadré, pas encore livré par l’IA.
- in-progress : feature réalisée par l’IA, prête à être testée/validée par l’humain (recette).
- done : validé par l’humain.

Déplacements
- Utiliser `git mv` pour changer de dossier (todo → in-progress → done) afin de conserver l’historique.

Nommage des tickets
- Format `YYYY-MM-DD_Mxx-slug.md` (ex: `2025-12-16_M13-validation-api.md`).

Rappel
- in-progress signifie “à tester par l’humain”, pas “en cours de dev”.
